Use Case & Objective

What & Why

What the agent does

Given three study topics — python_asyncio, agent_loops, sdk_rung2_patterns — the agent searches the web and fetches pages until every topic has at least two distinct source URLs. Then it stops.

🎯 The goal as code

The stopping condition is a Python function, not a sentence:

def is_coverage_met(coverage_map) → bool:
    # True when ALL topics have ≥2 URLs
💡
Why an agent and not a loop you'd write yourself? The model decides which topic to prioritise and whether to search or fetch based on what it observes mid-run. That model-owned decision is what makes this an agent.

① constants + MOCK_SEARCH_DB

Top of file. See STUDY_TOPICS, MAX_ITERATIONS = 10, and the mock result database. These are the agent's world.

② preflight()

Environment check. Verifies Python 3.12+, SDK import, CLI login. Fails fast with a fix-it message. Demonstrates goal-directedness — the agent won't start without preconditions met.

③ is_coverage_met()

Short function, huge concept. G1 — the goal as a predicate. Note it returns bool, not a string. That single difference separates an agent from a workflow.

④ search_web() + fetch_page()

Mock tools. Note the trust fence inside fetch_page() — external content is wrapped before it can reach the LLM prompt. It cannot be skipped.

⑤ call_llm()

The only function that touches claude_agent_sdk. It calls query() as an async iterator, collects AssistantMessage and ResultMessage, and returns the text + token counts. Mock this in smoke_test.py.

⑥ agent_loop() — the while loop

The heart of the agent. Read the OBSERVE / REASON / ACT comments. Notice how coverage_map is injected into the system prompt at the START of every iteration — this is how a stateless query() "sees" prior results.

⑦ print_summary()

Runs unconditionally at exit. Connects every trait and gate to specific output lines from the run that just completed.

Learning Insights

Key concepts

🔄 The loop is the unit of agency

A single call_llm() call is a workflow step — input in, output out. The while loop wrapping it is what makes it an agent: observe result → update coverage_map → new observation next iteration. Remove the loop and all you have is a smart search call.

🧠 State lives in Python, not the LLM

Every query() call in this agent is stateless — each starts a fresh LLM conversation with no memory of prior turns. coverage_map is Python state, re-injected into the system prompt at the start of every iteration so the model can "see" history. SDK rung 3 (session_id in ClaudeAgentOptions) changes this: it gives the LLM a persistent conversation thread so you no longer need manual re-injection.

📂 Your run log accumulates

study-prep_run_output.log is written in append mode — every run adds a timestamped section. This file is the primary input for study-prep_learning-insights.html (Part 2), generated after you run the agent.

🎲 The [MODEL DECISION] line is the agent/workflow boundary

That line marks the moment where the model decided which tool to call, not your code. Replace it with if topic == "x": search_web() and you'd have a workflow. The LLM choosing at runtime is what separates the two. G2 passes because of that one decision point.

Gotchas

Common traps
GOTCHA

Counting duplicate URLs inflates coverage

If search_web returns a URL already in coverage_map[topic], adding it again makes is_coverage_met() fire too early. Guard: if url not in coverage_map[topic]: coverage_map[topic].append(url)

GOTCHA

Hardcoding the tool sequence breaks G2

If your code always calls search_web first then fetch_page, the model's decision is overridden. G2 fails — it's a workflow masquerading as an agent. Parse the model's response; never pre-select the tool in Python.

GOTCHA

Treating a ConnectionError as structural

A network timeout is transient (temporary). Letting it propagate crashes the loop and loses all collected sources. Classify as ⚠ [ERROR: TRANSIENT], skip that URL, continue. Only halt on STRUCTURAL errors (bug in your code).

GOTCHA

ANTHROPIC_API_KEY routing bypass

If set in your shell, the SDK may route calls through the paid API instead of your Max plan CLI. preflight() detects and strips it from the child-process env only — never from the global shell.

GOTCHA

Testing only the happy path leaves G4 unverified

The cap-reached exit is just as important as goal-met. Without a test for it, a runaway loop bug goes undetected. Both exits need their own fixture in smoke_test.py.

📝
Why [PLAN REVISED] doesn't fire here: The study plan is linear — 3 fixed topics. The model searches them without needing to restructure. You'll first see [PLAN REVISED] in cycle 3, when the model encounters an unexpected tool result and rearranges the remaining steps on its own.

MCQ Self-Check

Test yourself
1. Why does the agent re-inject coverage_map into the system prompt at the start of every iteration?
Correct. Every query() is stateless — no memory of prior calls. coverage_map is Python-side state. Re-injection is how the model "observes" history. SDK rung 3 (session_id in ClaudeAgentOptions) replaces this pattern.
2. What would happen if you removed the duplicate-URL guard from the coverage_map update?
Correct. Without the guard, the same URL counted twice makes len(sources) >= 2 return True after only one real source. False positive goal-met.
3. What is the single clearest difference between this agent and a workflow that does the same task?
Correct. The 🎲 [MODEL DECISION] line is the proof: the model selects search_web or fetch_page based on current coverage state — a decision that couldn't be hardcoded in advance.
4. The agent runs 10 iterations and is_coverage_met() still returns False. Which exit fires?
Correct. The circuit breaker overrides the goal predicate. MAX_ITERATIONS = 10 is a hard cap — the agent exits with EXIT: cap reached regardless of coverage state. This is G4: principled termination requires BOTH a goal exit AND a hard cap.
5. Why is the trust fence applied inside fetch_page() rather than at the call site?
Correct. Applying the fence at the source (inside the function) means no caller can forget it. If the fence were at the call site, a future developer calling fetch_page() from a new code path might skip it, opening a prompt-injection vulnerability.

Takeaway & Ready-Gate

Before you run
🎓
Core takeaway from rungs 1+2: An agent is a goal-driven loop where the model owns at least one branching decision per iteration and prior observations change subsequent actions. The circuit breaker ensures it terminates. Remove any of these four properties and you have a workflow.
  • ✓I can describe what is_coverage_met() does and why it returns bool
  • ✓I understand that each query() is stateless — coverage_map must be re-injected each call
  • ✓I can name what the model decides vs what Python decides
  • ✓I know why both exit paths must be tested in smoke_test.py
  • ✓I understand what the trust fence prevents and where it lives
  • ✓I know where study-prep_run_output.log appears and what it's for
